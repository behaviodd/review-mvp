## 5. 함정 / 주의

- **saveAnswer 디바운스 800ms**: typing 직후 페이지 unmount 면 800ms 안에 sync 시도. 페이지가 떠나면 setTimeout 은 살아있지만 store 갱신은 ok (zustand persist). 만약 라우팅 race 가 의심되면 `flushAnswerSync` 를 cleanup 시점에 추가 고려
- **BroadcastChannel 범위**: 동일 origin + 동일 브라우저만. 다른 사용자/다른 브라우저는 grace 모름 (B7 잔여 한계)
- **ModalShell size 마이그레이션**: 기존 widthClass 16곳 그대로 작동. size 권장은 신규 코드에서. 일괄 마이그레이션은 별도 phase
- **parser round-trip 작동**: 운영 시트의 정책 컬럼이 빈 셀이면 여전히 fallback. P0-2 의 admin 재푸시로 채워야 의미
- **Apps Script 의 _계정 분기**: 클라이언트 호출은 차단됐지만 createUser 안의 _계정 upsert 는 여전. deploy 동반 정리 필요
- **handoff-2026-05-18.md 의 "현재 모드"**: 마이너 UI 디테일 → 이번 세션은 정합·안정성·QA 처리로 전환
