## 5. 인증 버그 3건 fix

### 5-1. `1119f73` — 새로고침 후 401 재발 방지

**문제**: `idToken`은 메모리 전용. 새로고침 후 `currentUser`는 localStorage 복원, `idToken`은 null → 인증 헤더 없이 sync 호출 → 401.

**이전 코드** (`AppLayout.tsx`):
```js
if (!idToken) return; // 아무 처리 없이 종료
```

**수정 후**:
```js
if (!idToken) {
  // setTimeout(0): login 직후 setIdToken/login 순서 차이 방어
  const t = setTimeout(() => { logout(); navigate('/login', { replace: true }); }, 0);
  return () => clearTimeout(t);
}
```

### 5-2. `be6f49f` — 로그인 자체 401 (닭-달걀 문제)

**문제**: `verifyGoogleLogin`이 `/api/org-sync` POST를 호출하는데, 로그인 전이라 `idToken`이 없어 인증 헤더 없음 → 401.

**수정**: `authApi.ts`의 `post()`에 `overrideToken` 파라미터 추가. `verifyGoogleLogin(credential)`에서 credential 자체를 Bearer로 전달 (credential = 유효한 Google ID Token이므로 `requireAuth()` 통과).

```js
// authApi.ts
export async function verifyGoogleLogin(idToken: string) {
  const json = await post('verifyGoogleLogin', { idToken }, idToken); // overrideToken = credential
```

### 5-3. `23f3bca` — 로그인 전 org-sync GET 401

**문제**: `OrgSyncProvider`는 `RequireAuth` 바깥 최상위 마운트 (Login 페이지에서 `users` 데이터 로드 필요). Phase 1이 GET도 인증으로 막으면서 로그인 전 org 데이터 로드 불가 → `users` 비어 있음 → 로그인 자체가 깨짐.

**수정**: `org-sync GET`만 `requireAuth()` 제외. POST(쓰기)는 유지.

```
org-sync  GET  → 인증 불필요 (로그인 전 members 로드 필요)
org-sync  POST → requireAuth() 필요
review-sync 전체 → requireAuth() 필요 (RequireAuth 안에서만 마운트)
```
