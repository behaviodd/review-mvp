## 2. 오늘 작업 — R8 자동 승인 모드

### 2-1. 배경 (사용자 의도)

배포 시 **기존 makestar 도메인 계정 보유 직원 N명의 일괄 로그인 가능화** 가 운영자 부담의 큰 덩어리. 현행 흐름은 첫 로그인 → 대기 큐 → 운영자 1건씩 승인 → active. N 명이면 운영자가 N 건 승인 작업 필요.

**선택한 해법**: 운영자 토글 자동 승인 (옵션 B-2). 배포 직후 1~2주 토글 ON 으로 전환해 사용자가 첫 로그인하는 즉시 활성화 → 안정화 후 OFF 로 전환해 정상 승인 큐로 복귀.

**미선택 옵션** (이유):
- B-1 영구 자동 승인 → 끄는 스위치 없어 회수 비용 ↑
- B-3 자가 보강 폼 결합 → 신규 UI 큰 작업, 일정 영향
- A. `_구성원` 시트 일괄 사전 등록 → Workspace Admin 권한자 데이터 협조 필요. **별도 진행** (사용자가 타 부서와 한 번에 모아서 처리하기로)

### 2-3. 안전 장치

1. **도메인 검증은 그대로 유지** — `verifyGoogleIdToken_` 안에서 `hd === ALLOWED_HD` + `email_verified === true` 검증. 자동 승인은 그 다음 단계에서만 동작
2. **rejected 이메일 차단 우선** — 자동 승인 분기 *위에서* `rejected` 케이스 차단. 거절된 사용자가 자동 등록되지 않음
3. **임시 사번 prefix `auto_`** — 운영자가 시트에서 즉시 식별 가능. 정식 사번(`YYYY###`) 과 prefix 충돌 없음
4. **권한그룹 미부여** — 일반 멤버 권한만. admin/leader 등 상승은 운영자 명시 부여 필요
5. **감사 로그 분리** — `member.auto_approved` (자동) vs `member.approved` (수동). 발생자ID = `system_auto`
6. **토글 변경도 감사** — `setting.changed` 액션. 누가 언제 ON/OFF 바꿨는지 추적
7. **default OFF** — `_설정` 시트 미존재 또는 키 없으면 `settingGet_` 이 default `'false'` 반환 → 자동 승인 OFF

### 2-4. 보강이 필요한 자동 승인 멤버 식별

자동 승인된 멤버는 다음 두 조건을 모두 만족함:
- 사번이 `auto_` 로 시작
- `주조직ID` 가 빈값

운영자 후처리 흐름:
1. `/team` 에서 "미배정" 그룹에 자동 승인 멤버들이 자동으로 노출됨 (orgUnitId 빈값 필터링)
2. 멤버 클릭 → 정식 사번 / 직책 / 직무 / 조직 / 보고대상 채움
3. 시트 직접 편집 또는 `updateUser` API 사용
4. **사번 변경 함수는 별도 phase 필요** — 현재 `updateUser` 가 사번 변경을 지원하는지 검증 안 됨. 사번 = PK 라 변경 시 권한그룹/평가권자 모두 갱신 필요. 이 부분은 § 4 잔여 작업으로 분리

## 5. 함정 / 주의

1. **`_설정` 시트는 자동 생성** — `getSheet(SHEET_SETTINGS, SETTINGS_HEADERS)` 가 처음 호출 시 만듦. 첫 호출 전엔 시트 없음. 헤더는 자동 보강
2. **자동 승인 분기는 `_invalidateBulkCache_()` 명시 호출 필요** — `verifyGoogleLogin` 은 BULK_AFFECTING_ACTIONS_ 화이트리스트 밖에서 실행되어 자동 캐시 무효화가 안 걸림. 자동 승인 분기에서 명시적으로 호출함 (L724)
3. **`처리자 = 'system_auto'` 마커** — `대기승인` 시트의 자동 승인 row 식별용. 기존 운영자 사번 자리에 들어감 — 시트에서 운영자가 보고 식별 가능
4. **클라이언트 토글 카드는 `scriptUrl && admin` 모두 만족 시만 노출** — 시트 미연결 환경에서는 토글 자체가 의미 없음. fetch 실패 시 `false` 로 fallback (UI 표시는 OFF)
5. **auto_approve_domain 값은 문자열 'true' / 'false'** — `=== 'true'` 엄격 비교. 'True', '1' 등은 OFF 로 처리됨. 운영자가 시트 직접 편집 시 주의 (UI 사용 권장)
