### 2-5. 구성원 미표시 버그 수정 (`panelUsers` 레거시 매칭 개선)

**근본 원인**: 특정 org를 클릭했을 때 "구성원이 없습니다."가 반복 표시됨.

조직유형 fallback 로직에 의해 `parentId` 있는 조직이 `type = 'team'`이 되는데, 사용자가 `_구성원` 시트에서 `주조직` 컬럼만 채우고 `팀` 컬럼은 비워둔 경우 레거시 매칭이 `u.team` 필드를 확인 → `undefined` → 0명.

**수정** (`Team.tsx:866`):
```typescript
const legacyIds = activeUsers
  .filter(u => {
    const primary = u[legacyKey[selectedUnit.type]];
    if (primary === selectedUnit.name) return true;
    // 주조직(department)만 입력한 경우 — subOrg/team/squad 타입 조직에서도 매칭
    if (selectedUnit.type !== 'mainOrg' && u.department === selectedUnit.name) return true;
    return false;
  })
  .map(u => u.id);
```

**빈 상태 메시지 개선** (`Team.tsx:1111`): `전체` 뷰에서 users는 있지만 activeUsers가 비었을 때 "활성 구성원이 없습니다. (전체 N명 중 비활성)" + `상태분류` 컬럼 확인 안내.

## 5. 알아둘 사항 / 함정

- **`deriveOrgUnitsFromMembers`의 타입**: 파생 조직은 항상 `type: 'mainOrg'`. `_조직구조` 시트 조직명과 `주조직` 값이 다르면 파생 조직이 별도로 생김 (이름 불일치 → 두 조직이 트리에 공존). 사용자가 `_조직구조`를 클릭했을 때 구성원이 안 보이면 이 이름 불일치가 원인일 가능성 높음.
- **`panelUsers` 레거시 매칭 폴백 범위**: `department` 폴백은 `selectedUnit.type !== 'mainOrg'`인 경우만. mainOrg는 이미 `department` 필드를 primary로 체크.
- **`reviewerAssignments` store**: store 자체는 남아있으나 UI에서 완전 미사용. 시트에서 읽어온 값은 store에 있지만, 표시·변경 로직은 모두 `managerId` 기준.
- **`상태분류` 컬럼**: `active` / `leave_short` / `leave_long` / `terminated` / `other` 영문 값만 인식. 한국어(`재직중` 등) → `activityStatus = undefined` → `isActive` 필드 fallback (기본 true). 별도 한국어 alias 미구현.
