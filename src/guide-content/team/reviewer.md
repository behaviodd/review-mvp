## 개요

평가권자는 사이클에서 **하향 평가**(downward) 를 작성할 사람을 말합니다. 1차(직속), 2차 등 차수별로 명시적으로 배정합니다. 1차는 보고대상(`managerId`) 이나 소속 조직의 조직장(`headId`) 으로 자동 fallback 되지만, 명시적 배정이 우선합니다.

## 단계별 안내

### 1) 평가권자 모델 *(현재 표시 위치)*

현재 구성원 drawer 는 평가권자 데이터를 **권한 계산용으로만** 사용합니다 (필드 가시성·작성 권한). 차수별 평가권자 배정 현황을 보여주는 별도 시각 섹션은 추후 추가 예정입니다.

차수의 의미는 다음과 같습니다:

- **1차(직속)** — 자동 fallback 또는 명시 배정. 명시 배정이 없으면 보고대상(`managerId`) → 소속 조직의 조직장(`headId`) 순으로 자동 매핑
- **2차** — 명시 배정만 인정 (자동 fallback 없음)

### 2) 평가권자 배정·변경 *(예정 기능)*

> 평가권자 데이터 모델(`ReviewerAssignment` — 차수 / 시작일·종료일 / 출처) 과 사이클 발행 시 적용 로직은 이미 동작합니다. 다만 **구성원 페이지 drawer 안에서 직접 배정·변경하는 모달 UI 는 아직 추가되지 않았습니다**. 현재 단계에서는 운영자가 시트의 `_평가권자배정` 탭에서 직접 row 를 추가합니다. UI 추가는 차기 작업에서 진행됩니다.

배정에 들어가는 항목 (모달 추가 시 입력 예정):

- **차수** — 1차 / 2차 / 3차 ... 선택
- **평가권자** — 활성 멤버 중 선택
- **시작일·종료일** — 배정 효력 구간

### 3) 차수 추가 (2차 이상)

조직장 평가를 1차+2차 모두 받고 싶을 때:

1. **사이클 편집** 에서 `downwardReviewerRanks: [1, 2]` 로 설정
2. 시트 `_평가권자배정` 에서 멤버별로 2차 평가권자를 명시적으로 추가 (자동 fallback 없음)
3. 사이클 발행 시 사전 점검에서 **2차 평가권자 미배정** 차단이 뜨면 누락된 멤버를 확인

## 자주 묻는 질문

**Q. 평가권자가 admin role 사용자여도 되나요?**
A. 네. admin 도 평가권자가 될 수 있습니다. 인사팀장이 직접 평가하는 케이스가 흔합니다.

**Q. 1차 평가권자를 비우고 2차만 두면 어떻게 되나요?**
A. 사이클 발행 시 `downwardReviewerRanks: [2]` 로 설정하면 가능합니다. 사전 점검에서 2차 미배정 멤버만 차단됩니다.

**Q. 평가권자가 퇴사하면 진행 중 사이클은 어떻게 되나요?**
A. 진행 중 사이클은 **발행 시점의 평가권자가 동결** 되어 그대로 유지됩니다. 작성 자체는 가능합니다 (제출은 정책에 따라). 다음 사이클 발행 전 멤버별로 새 평가권자를 배정해야 사전 점검을 통과할 수 있습니다.

**Q. 보고대상(managerId) 변경과 평가권자 배정 중 어느 게 우선인가요?**
A. 명시적 ReviewerAssignment(rank=1) 가 우선입니다. 보고대상은 ReviewerAssignment 가 없을 때만 fallback 으로 사용됩니다. 두 정보를 일치시켜 두는 것이 운영상 깔끔합니다.
