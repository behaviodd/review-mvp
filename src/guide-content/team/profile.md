## 개요

구성원 페이지에서 멤버 행을 클릭하면 우측 drawer 가 열려 정보를 확인하고 수정할 수 있습니다.

## 단계별 안내

### 1) 구성원 페이지 열기

좌측 메뉴 **구성원 관리 › 구성원**.

![구성원 페이지 — 좌 멤버 / 우 조직도](/guide-images/team/60-team-page.png)

화면은 좌우로 나뉩니다:
- **좌측** — 멤버 리스트 (검색·필터·정렬 가능)
- **우측** — 조직도 트리 (조직 추가/이동/삭제 — [조직 편성](/guide/team/org-tree) 참고)

### 2) 멤버 drawer 열기

멤버 행을 클릭하면 우측에 drawer 가 열립니다. 기본 정보, 소속 조직, 평가권자 정보를 확인할 수 있습니다.

![멤버 drawer 진입 화면](/guide-images/team/61-member-drawer.png)

### 3) 정보 수정

drawer 우상단 **편집** 버튼을 누르면 입력 모드로 전환됩니다. 수정 가능 항목:

- 이름·이메일 (이메일 변경은 SSO 와의 정합 주의)
- 직책·직무
- 소속 조직 (드롭다운에서 변경)
- 보고대상(1차 매니저)
- 활성/휴직/퇴사 상태

![멤버 정보 편집 모드](/guide-images/team/62-member-edit.png)

**저장** 을 누르면 즉시 시트로 동기화됩니다.

### 4) 활성 상태 변경 *(예정 기능)*

> 활성/휴직/퇴사 상태(`activityStatus`) 는 데이터 모델과 drawer 표시는 이미 반영되어 있으나, **편집 폼에 상태 드롭다운은 아직 추가되지 않았습니다**. 현재 단계에서는 운영자가 시트에서 직접 상태를 변경합니다. UI 추가는 차기 작업에서 진행됩니다.

상태 값 정의:

- **활성(active)** — 일반 근무
- **단기 휴직(leave_short)** — 사이클 대상에 포함됨 (출산휴가 등)
- **장기 휴직(leave_long)** — 사이클 대상에서 자동 제외 (육아휴직, 안식년 등)
- **퇴사(terminated)** — 사이클 대상에서 자동 제외 + 로그인 차단

drawer 상단에는 현재 상태가 배지로 표시됩니다 (`62-member-edit.png` 참고).

## 자주 묻는 질문

**Q. 보고대상을 바꾸면 진행 중 사이클의 평가권자도 바뀌나요?**
A. 진행 중 사이클은 **발행 시점의 평가권자가 동결** 되어 영향받지 않습니다. 다음 사이클 발행부터 새 보고대상이 반영됩니다.

**Q. 퇴사 처리한 멤버의 과거 리뷰는 어떻게 되나요?**
A. 그대로 보존됩니다. 받은 리뷰·작성한 리뷰 모두 시트에 유지됩니다. 다만 다음 사이클 발행 대상에서는 자동 제외됩니다.

**Q. 이메일을 바꾸면 SSO 로그인은 어떻게 되나요?**
A. SSO 는 Google 의 이메일을 기준으로 매칭됩니다. 시트의 이메일을 바꾸면 다음 로그인 시 사용자가 다시 대기승인 큐에 들어갈 수 있으니 주의가 필요합니다.
