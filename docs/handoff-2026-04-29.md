## 8. 알려진 함정·주의사항

어제 § 10 의 8개 항목 + 본 세션 추가 3개:

1. **`useReviewSync`/`useOrgSync` 는 App.tsx 의 OrgSyncProvider/ReviewSyncProvider 에서만 mount** — 다른 곳에서 다시 호출하면 polling 중복.
2. **resilientFetch 의 4xx 는 permanent throw** — 호출자는 별도 throw 불필요. 단 Apps Script 의 body-level error (200 응답 + `{error: ...}`) 는 별도 throw 필요.
3. **markPendingWrite 는 두 번 호출이 정상** (fetch 전·후, sheetWriter.ts 주석 참조).
4. **POST 는 멱등 액션만 retry** — `IDEMPOTENT_ACTIONS` 세트 외 액션은 retries=0.
5. **인사스냅샷 시트** 가 시점 의존 조직도의 단일 진실.
6. **Vercel Edge functions 는 별도 배포** — `npm run build` 는 검증하지 않음.
7. **bulkGetAll 미배포 폴백** — Apps Script 가 미지원이면 6개 병렬 fetch 폴백.
8. **`LockService`** — 본 세션에서 도입 (`withLock_`). doPost 진입 시 자동 획득. 6분 한계 주의 (4-4 #2).
9. **`fadbc0e` Apps Script 미배포** — 디자인 작업 중 배포 보류 결정. § 4-2 참조.
10. **audit dedupe 5초 버킷** — `auditLogStore.deriveAuditId` 의 윈도우. 사용자 빠른 재시도 (~0.5~3s) 흡수 + 의도적 두 번째 액션과 분리. 다중 탭 사용 빈도 측정 후 조정 가능 (§ 4-4 #4).
11. **`withLock_` 의 lock_busy 응답** — 200 OK + body 에 `error:'lock_busy'`. status code 로는 못 구분 (Apps Script ContentService 제약). client 의 IDEMPOTENT 자동 retry 로 흡수.
